暂存（add）与提交（commit）

如图当目录中新建了文件之后，可以在右侧的Unstaged changes窗口看到变化。绿色+表示添加内容

下面将文件暂存git add main.c，Lazygit的快捷键是按键空格或a

  • 空格键只暂存当前选中的文件

  • a暂存所有Unstaged change（.gitignore中指定的除外）

注：

  • Lazygit的快捷键多数是git指令的第一个字母，并且支持用户在~/.config/lazygit目录自定义快捷键

  • 按下快捷键时请注意观察右下角Command log窗口，会同步显示完整的git指令

按下空格键暂存`main.c`的效果，标记为A

将暂存的修改提交到本地仓库git commit -m "add main.c"，Lazygit快捷键是c

  • Commit summary：提交信息，与git log --oneline效果相同

  • Commit description：建议将详细的提交信息写在此处，之后可通过编辑器或Log窗口查看，避免提交窗口字符过多，与git log效果相同

填好后在Commit summary窗口按回车键即可提交

提交`main.c`后的效果

如图所示，刚刚填写的提交历史ID简写、提交用户名简写和Commit summary、显示在左侧[4]Commit窗口

详细的提交信息显示在右侧Log窗口与git log效果相同。

修改（amend）

  • 修改文件内容

如图所示，当用户修改文件后，绿色+表示添加内容，红色-表示删除内容。

上面演示添加时是在文件的级别上操作，如果只想暂存部分修改内容，可以进入右侧Unstaged changes窗口选择。

此处涉及到窗口切换，在此介绍一下窗口切换的快捷键：

  • 窗口编号为1-5，可以使用数字键切换

  • h,l在5个窗口间切换

  • [,]在每个窗口的子窗口切换

  • j,k在每个窗口中光标上下移动

  • 回车进入右侧窗口，例如在选中main.c时按回车进入右侧Unstaged changed窗口

  • +,-,++,--放大或缩小某个窗口，（按3次会缩小回原状态）

下面选中main.c时按回车进入右侧Unstaged changed窗口，如图

这时会显示Unstaged changes和Staged changes两个窗口（Tab键在两个窗口间切换），j,k上下移动，空格选中之后要暂存的内容（选中的内容会显示在Staged changes）

选好之后按键返回[2]Files窗口，按c键并填写commit信息即可提交，效果如下图

在[4]Commits窗口查看修改（行号，添加删除内容等）

如果这时还想修改main.c文件，并且为了避免Commits log过长，把新修改amend到刚刚的commit上：

回到[2]Files窗口，按e进入编辑器修改main.c文件，空格键选中main.c文件，按A键并回车

进入[4]Commits查看，发现两次提交都合并到了C | main.c提交，如果想修改commit信息按r键即可重写commit信息

上面的修改仅是单个文件，如果是同时提交多个文件方法相同，不过为了单独查看每个文件的修改，需要用到Diff窗口

如图，添加了两个文本文件，右侧Patch窗口同时显示了所有文件的修改，按下回车键，进入Diff窗口，按+放大，效果如下图所示，可分别查看一次提交的每个文件的修改，j,k在各文件间跳转，e进入编辑器修改内容

如果发现某个修改不想保留，按d删除

版本回退（reset）

按g键回退到在[4]Commits窗口选中的提交ID（首字母与reset不同，r键已被绑定在reword指令）

如图，有三种reset方式可供选择，与git reset原理一致。比如选择soft模式，效果如下图

如果发现不是想回退到这个版本，按]进入Reflog窗口重新选择

这里显示了commit历史记录，回车可进入内部查看所有修改，空格选中某个版本git checkout。

分支管理与冲突

进入[3]Local branches窗口按n键新建分支，选中某个分支，空格键即可checkout到对应分支

使用git时会涉及到两个用户在各自创建的分支修改同一个文件的情况，比如：

合并分支方法（例如将Alice分支合并到Bob分支）：首先在[3]Local branches窗口按空格checkout到Bob分支，然后j,k切换到待合并的子分支，按M合并

Alice和Bob同时修改了b.txt的同一行，此时如果想把Alice分支合并git merge到Bob分支会发生冲突，提示如图

回车键查看冲突如下图所示，红色HEAD指针指向Bob分支，Alice指针指向Alice分支，右侧Merge conflicts窗口显示了所有的冲突，回车进入右侧窗口

j,k上下移动光标，空格选中要保留的内容，当所有冲突被解决时，被提示继续merge，如下图所示

合并结束后在[4]Commits窗口可以图形化显示分支合并的状态，右侧显示整体修改情况，回车进入也可以查看具体的修改内容。

挂起（stash）

git stash 可以临时保存当前工作目录的修改，然后切换到其它分支。

例如在Eve分支创建了一个新文件eve.txt，但是还不想提交，可以空格选中该文件，按s键stash，如下图

此时切换到其它的分支，也可以在[5]Stash窗口看到刚刚挂起的内容

想在任何一个分支处理这个挂起，切换到[5]Stash窗口

  • 空格键（Apply）：将挂起内容放到[2]Files窗口，表示未暂存状态

  • d（Drop）：直接删除

  • p（Pop）：与Apply的区别是会删除暂存区的更改

标签（tag）

当某次提交很重要时，可以加个标签，比如v1.0，git指令为git tag

在Lazygit中进入[3]Tags窗口，按n键创建标签

如图，可以发现最新的一次提交被加了标签，在[3, 4]窗口均可查看。
